Outline

Project Root Folder <- Root of the project
  • docker-compose.yml <- Orchestration file that contains the configuration to develop locally, it overrides the Dockerfile to run the flask development server instead of running the gunicorn server that is used for hosting on Heroku
  • Dockerfile <-File used to create an image to run in a container that runs the program
  • heroku.yml <-It allows you to build docker images on heroku and also to specify add-ons and configure variables which can created during app provisioning
  • pytest.ini <-This file consists of the pytest configuration if it has the pytest section. It is the main config file which can make changes in the running mode of pytest.
  • readme.md <-It's a text file that provides information for the user about the project, as well as instructions, assistance, and information on patches and upgrades.
  • requirements.txt <-It is a sort of file that typically contains information on all the libraries, modules, and packages that are utilized in the development of a project. It also keeps track of all the files and packages that the project depends on or needs to execute.
  • setup.py <-It indicates that a module or package has likely been packaged and distributed with it.
.github Folder <- The .github folder contains Workflows, issue templates, pull request templates and other project-specific files are all stored under the. github directory.
workflows Folder <- The workflows folder contains Workflows of the project.
  • dev.yml <- This contains all the environment variable, command line arguments, properties used for development.
  • prod.yml <- This contains all the environment variable, command line arguments, properties used for production.
app Folder <- The app folder contains all the html,js,css or images required for the project.
simple_pages Folder <- The folder contains all the html and _init_.py files required for the project.
templates Folder <- The templates folder contains all the html files required for the project.
  • index.html <-This is the homepage of the project.
  • Docker.html <-This contains information about the GIT.
  • Git.html <-This contains information about the DOCKER.
  • python-flask.html <-This contains information about the PYTHON/FLASK.
  • ci-cd.html <-This contains information about the CI/CD.
static Folder <- The static folder contains all the css, images and js files required for the project.
  • css folder <-It contains all the css files required for the project.
  • images folder <-It contains all the images required for the project.
  • js folder <-It contains all the js files required for the project.
templates Folder <- The templates folder contains the base html file required for the project. test Folder <- The test folder contains the all the testing files required for the project.
  • simple_pages_tests.py <-It contains all the required test cases required for the project.
